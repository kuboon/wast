# cargo-component Reference (v0.21.1)

## Installation

```bash
cargo install cargo-component --locked
```

## Building Components

```bash
cargo component build                        # Build all component crates
cargo component build -p wast-partial-manager # Build specific crate
cargo component build --release               # Release build
```

This compiles to `wasm32-wasip1` target and creates component .wasm files in `target/wasm32-wasip1/{debug,release}/`.

## Cargo.toml Configuration

### Component crate (cdylib)

```toml
[package]
name = "wast-partial-manager"
version.workspace = true
edition.workspace = true

[dependencies]
wit-bindgen = { workspace = true }
wit-bindgen-rt = { workspace = true }  # REQUIRED: runtime support for generated bindings

[lib]
crate-type = ["cdylib"]

[package.metadata.component]
package = "wast:core"

[package.metadata.component.target]
path = "../../wit"            # Path to shared WIT directory (relative to crate root)
world = "partial-manager-world"  # World name from the WIT file
```

### Library crate (not a component)

Regular `[lib]` crate. No `[package.metadata.component]` section. No `wit-bindgen` dependency needed.

## Generated Bindings

cargo-component auto-generates `src/bindings.rs` at build time. **Do not edit this file.**

The generated file contains:
- Type definitions matching WIT records, variants, enums
- Guest traits for exported interfaces
- `export!` macro for wiring up the implementation

```rust
    fn extract(

        // implementation
    }

    fn merge(
        partial: bindings::wast::core::types::WastComponent,
        full: bindings::wast::core::types::WastComponent,
    ) -> Result<bindings::wast::core::types::WastComponent, Vec<bindings::wast::core::types::WastError>> {
        // implementation
    }
}

bindings::export!(Component with_types_in bindings);
```

### Type Path Convention

- Exported interface traits: `bindings::exports::{namespace}::{package}::{interface}::Guest`
- Imported types: `bindings::{namespace}::{package}::{interface}::{TypeName}`
- For this project: `bindings::wast::core::types::WastComponent`, etc.

## Shared WIT in Workspace

All crates point to the same `wit/` directory via relative paths in `[package.metadata.component.target]`:

| Crate depth | path value |
|---|---|
| `crates/partial-manager/` | `../../wit` |
| `crates/syntax-plugin/ruby-like/` | `../../../wit` |

## Common Pitfalls

1. **Missing `wit-bindgen-rt`**: The generated bindings reference `wit_bindgen_rt::run_ctors_once()`. You MUST add `wit-bindgen-rt` as a dependency.
2. **WIT keyword conflicts**: Use `%` prefix for reserved words in WIT files (e.g., `%u32`, `%bool`, `%option`).
3. **Regenerating bindings**: Run `cargo component build` after changing WIT files — it regenerates `src/bindings.rs`.
4. **Do not check in bindings.rs**: Add `src/bindings.rs` to `.gitignore` if desired (it's auto-generated).
